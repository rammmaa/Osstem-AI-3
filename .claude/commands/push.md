변경사항을 자동으로 commit하고 push하는 작업을 수행해 주세요.

## 절차

1. `git status`로 변경사항 확인
2. 변경사항이 없으면 "변경사항이 없습니다"라고 알려주고 종료
3. 변경사항이 있으면:
   - `git diff`와 `git diff --staged`로 변경 내용 분석
   - 변경 내용을 바탕으로 한국어 커밋 메시지 자동 생성 (형식: `타입: 설명`)
     - 타입 예시: feat, fix, docs, refactor, style, chore
   - 보안 체크: .env 파일이나 API 키가 포함된 파일이 있으면 경고하고 해당 파일은 제외
   - `git add`로 변경된 파일 스테이징 (.env, 자격증명 파일 제외)
   - 생성된 메시지로 `git commit` 실행
   - `git push origin` 으로 현재 브랜치에 push
4. 결과를 간단히 요약해서 알려주세요

## 주의사항

- .env 파일은 절대 커밋하지 마세요
- 실제 API 키나 비밀번호가 하드코딩된 파일이 있으면 경고하세요
- push 실패 시 원인을 알려주세요
