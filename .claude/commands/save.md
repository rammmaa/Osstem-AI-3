## 1단계: CLAUDE.md 업데이트

1. 현재 프로젝트 상태 파악
   - `git log --oneline -10`으로 최근 커밋 확인
   - 프로젝트 루트의 폴더/파일 구조 확인 (`ls`)
   - `.claude/` 디렉토리 내 파일 확인
   - `.claude/commands/` 디렉토리의 슬래시 커맨드 목록 확인

2. `.claude/CLAUDE.md` 파일을 읽고 아래 섹션들을 현재 상태에 맞게 업데이트
   - **구조**: 새로운 폴더나 파일이 추가되었으면 반영
   - **문서 현황**: 새로운 문서가 추가되었으면 표에 추가
   - **Claude Code 설정 (.claude/)**: 새로운 설정 파일이나 커맨드가 추가되었으면 반영
   - **슬래시 커맨드**: 새로운 커맨드가 추가되었으면 표에 추가
   - **진행 현황**: 오늘 날짜로 새로운 작업 내역 추가 (날짜 형식: YYYY-MM-DD)
   - 기존 내용은 삭제하지 말 것 (진행 현황은 누적 기록)
   - 추측하지 말고 실제 파일/폴더를 확인한 내용만 반영

## 2단계: Commit & Push

1. `git status`로 변경사항 확인
2. 변경사항이 없으면 "변경사항이 없습니다"라고 알려주고 종료
3. 변경사항이 있으면:
   - `git diff`와 `git diff --staged`로 변경 내용 분석
   - 보안 체크: .env 파일이나 API 키가 하드코딩된 파일이 있으면 경고하고 해당 파일 제외
   - 변경 내용을 바탕으로 한국어 커밋 메시지 자동 생성 (형식: `타입: 설명`)
     - 타입 예시: feat, fix, docs, refactor, style, chore
   - `git add`로 변경된 파일 스테이징 (.env, 자격증명 파일 제외)
   - 생성된 메시지로 `git commit` 실행
   - `git push origin`으로 현재 브랜치에 push
4. 결과를 간단히 요약해서 알려주세요

## 주의사항

- .env 파일은 절대 커밋하지 마세요
- 실제 API 키나 비밀번호가 하드코딩된 파일이 있으면 경고하세요
- push 실패 시 원인을 알려주세요
