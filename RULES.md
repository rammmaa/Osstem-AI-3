# 3조 프로젝트 규칙 & 초보자 가이드

## 이 프로젝트는 뭔가요?

우리 조가 맡은 업무를 **컴퓨터가 대신 해주는 프로그램**을 만드는 프로젝트입니다.
예를 들어, 매일 엑셀에서 복사-붙여넣기 하던 일을 버튼 하나로 끝내는 것이 목표입니다.

---

## 폴더 구조 — "서류 캐비넷"이라고 생각하세요

프로젝트 폴더는 **서류 캐비넷의 서랍**과 같습니다.
각 서랍에는 정해진 종류의 서류만 넣습니다.

```
Osstem-AI-3/
│
├── demos/                ← "시연용 서랍" — 발표할 때 보여줄 데모 앱
│   ├── task01_과제명/    ← 과제 1번 데모
│   ├── task02_과제명/    ← 과제 2번 데모
│   └── README.md         ← 데모 실행 방법 설명서
│
├── examples/             ← "실제 코드 서랍" — 진짜 업무 자동화 코드
│   └── task01-이름/      ← 과제 1번 자동화 코드
│
├── README.md             ← "표지" — 이 프로젝트가 뭔지 한눈에 보는 문서
├── RULES.md              ← "규칙집" — 지금 읽고 있는 이 문서
├── CLAUDE.md             ← "AI 설명서" — AI에게 우리 프로젝트를 알려주는 문서
├── .gitignore            ← "반입 금지 목록" — GitHub에 올리면 안 되는 파일 목록
└── .env.example          ← "비밀번호 양식" — 비밀번호를 어디에 적을지 알려주는 샘플
```

---

## 필수 파일 설명

### README.md — "프로젝트 표지"

책의 표지처럼, 이 프로젝트가 뭔지 처음 보는 사람도 알 수 있게 적는 문서입니다.

들어가야 할 내용:
- 우리 조 이름과 멤버
- 어떤 과제를 맡았는지
- 어떻게 실행하는지

### .gitignore — "GitHub 반입 금지 목록"

GitHub에 올리면 **안 되는 파일**을 적어두는 목록입니다.

비유하면 공항의 반입 금지 물품 안내문과 같습니다.

올리면 안 되는 것들:
- `.env` — 비밀번호가 들어있는 파일
- `venv/` — 내 컴퓨터에서만 쓰는 설치 폴더
- `__pycache__/` — 파이썬이 자동으로 만드는 임시 파일
- `.DS_Store` — 맥에서 자동으로 생기는 파일

### CLAUDE.md — "AI에게 주는 메모"

Claude Code(AI 코딩 도구)가 우리 프로젝트를 이해하도록 적어두는 메모입니다.

비유하면 새로 온 인턴에게 주는 업무 인수인계 문서와 같습니다.

들어가야 할 내용:
- 프로젝트가 뭔지 한 줄 설명
- 폴더 구조
- 코딩 규칙 (한국어 주석, 파이썬 3.11 이상 등)

### .env.example — "비밀번호 양식지"

실제 비밀번호는 안 적고, **어떤 비밀번호가 필요한지**만 알려주는 샘플입니다.

```
# 이렇게 생겼습니다
ECOS_API_KEY=여기에_한국은행_API키_입력
CLAUDE_API_KEY=여기에_Claude_API키_입력
```

실제 비밀번호는 `.env` 파일에 적고, 이 파일은 절대 GitHub에 안 올립니다.

## 데모 앱 3종 — "같은 요리, 다른 그릇"

하나의 과제를 **3가지 방식**으로 만듭니다.
같은 요리를 종이접시, 도자기, 도시락통에 담는 것과 비슷합니다.

### 1. HTML 파일 (`demo.html`) — "종이접시"

- 파일을 **더블클릭**하면 바로 브라우저에서 열립니다
- 아무것도 설치할 필요 없습니다
- OT 발표할 때 가장 간편합니다

### 2. Streamlit (`app.py`) — "도자기 접시"

- 웹 브라우저에서 예쁘게 보여주는 방식입니다
- 터미널에서 `streamlit run app.py` 입력하면 실행됩니다
- 그래프나 표를 인터랙티브하게 보여줄 수 있습니다

### 3. Tkinter (`main.py`) — "도시락통"

- 윈도우 프로그램처럼 실행되는 데스크탑 앱입니다
- `.exe` 파일로 만들어서 다른 사람에게 줄 수 있습니다
- 파이썬 없어도 실행 가능하게 패키징할 수 있습니다

## 보안 규칙 — "비밀번호는 금고에"

비밀번호나 API 키를 코드에 직접 쓰면 **절대 안 됩니다**.

이유는 간단합니다:
- GitHub는 누구나 볼 수 있습니다
- 코드에 비밀번호를 적으면 전 세계에 공개하는 것과 같습니다
- 한 번 올라가면 삭제해도 기록이 남습니다

**올바른 방법:**

| 이렇게 하세요 | 이렇게 하지 마세요 |
|---------------|-------------------|
| `.env` 파일에 비밀번호 저장 | 코드에 직접 `password = "1234"` |
| `.gitignore`에 `.env` 추가 | `.env` 파일을 GitHub에 올리기 |
| `.env.example`로 양식만 공유 | 카톡/메일로 비밀번호 공유 |

## 작업 순서 — "이 순서대로 하면 됩니다"

1. **README.md 작성** — 우리 조 소개, 멤버, 과제 목록 적기
2. **CLAUDE.md 작성** — AI가 참고할 프로젝트 설명 적기
3. **.gitignore 만들기** — 반입 금지 목록 설정
4. **과제 정의서 작성** — 멤버별 담당 과제와 계획 정리
5. **demos/ 폴더 만들기** — 과제별로 폴더 만들고 HTML 데모부터 시작
6. **examples/ 폴더 만들기** — 실제 자동화 코드 작성
7. **테스트** — 다른 사람 컴퓨터에서도 잘 되는지 확인

## 자주 쓰는 명령어 — "이것만 알면 됩니다"

```bash
# GitHub에서 프로젝트 받기 (처음 한 번만)
git clone https://github.com/rammmaa/Osstem-AI-3.git

# 내가 바꾼 파일 확인하기
git status

# 바꾼 파일 올릴 준비하기
git add 파일이름

# 변경사항 저장하기 (커밋)
git commit -m "무엇을 바꿨는지 한 줄 설명"

# GitHub에 올리기
git push

# 데모 실행하기 (Streamlit)
streamlit run app.py

# 파이썬 코드 실행하기
python main.py
```
